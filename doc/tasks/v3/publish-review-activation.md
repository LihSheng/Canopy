# Publish Review And Activation

## Goal

Let the user validate a cleaned version and activate one published result for
downstream ontology-ready use.

## Tasks

- [ ] Build publish review summary data.
- [ ] Show warning counts, required field coverage, and active version state.
- [ ] Block publish when required mappings are missing.
- [ ] Activate exactly one cleaned version at a time.
- [ ] Preserve older cleaned versions for audit and replay.

## Testing

- [ ] Add backend tests for publish validation rules.
- [ ] Add backend tests for active-version switching.
- [ ] Add UI tests for the publish confirmation state.
